# QQ聊天机器人 "知慧"

一个基于 LangChain 的智能 QQ 聊天机器人，角色为高中女生"知慧"，具备多维度情绪系统、多轮记忆管理、工具调用和图片识别功能。

## 功能特性

- **智能对话**：基于 LangChain Agent，支持工具调用（天气、新闻、搜索）
- **情绪系统**：四层情绪模型（快速衰减 → 半衰期 → 遗忘/锚点保护 → 基底保底），6 种情绪标签，感染映射，突变检测，和解映射
- **记忆管理**：短期记忆（SQLite）+ 长期记忆（ChromaDB 向量存储）
- **图片识别**：集成通义千问视觉模型，延迟并行识别，上限 3 张防刷 token
- **好感度系统**：7天半衰期衰减，根据用户情绪 + AI自身情绪拉动调整，后台漂移
- **状态查询**：支持 `/状态`、`/status` 等指令查看当前情绪状态（自然语言回复）
- **智能延迟回复**：主延迟 6 秒 + 分段字符延迟（第一段无字符延迟）
- **智能消息分段**：根据句号、省略号、颜文字边界自动拆分长回复
- **CQ码清理**：自动将 QQ 表情/图片等 CQ 码转为可读文本后再送入 LLM
- **社交过滤器**：傲娇人格表达层过滤 + 认知资源分配权重 + 好感度伪装策略
- **共情系数**：好感度影响感染映射强度，高好感更在意用户情绪
- **情绪偏置检索**：强烈情绪时优先检索情绪一致性记忆（委屈时更容易"翻旧账"）
- **认知偏置指令**：通过 system message 注入情绪影响信息解读倾向
- **管理员系统**：管理员绕过黑白名单，支持 `/admin` 调试指令
- **概率预检查**：概率检查提前到 LLM 调用之前，节省 API 成本
- **白名单/黑名单**：灵活控制私聊和群聊范围（空集合=不限制）

## 快速开始

```bash
cp .env.example .env
# 编辑 .env 填入 API 密钥和配置
pip install -r requirements.txt
python main.py
```

服务监听 `http://0.0.0.0:8000`，NapCat 配置 webhook 到 `http://<wsl-ip>:8000/webhook`。

## 核心机制

### 情绪系统（四层模型）

情绪系统使用 6 个标签：**开心、生气、委屈、害羞、醋意、撒娇**（外加"中性"）。每个情绪值范围 0~10（浮点数），允许多个情绪同时存在。

#### 第一层：快速衰减层（对话间隙回落）
每条新消息到来时，如果距上条消息不超过 30 分钟，先将所有情绪 × 0.9，再执行遗忘检查。超过 30 分钟则跳过，仅依赖半衰期。

#### 第二层：半衰期层（持久情绪）
不同情绪有不同半衰期，随真实时间消退：
- 生气：30 分钟 | 醋意：40 分钟 | 委屈：2 小时
- 害羞：1.5 小时 | 开心：1 小时 | 撒娇：1 小时

公式：`factor = 0.5 ^ (经过秒数 / 半衰期)`

#### 遗忘机制与锚点保护
快速衰减后执行遗忘检查：强度 ≤ 2 且高于基底的情绪有 20% 概率归零。
**锚点保护**：曾达到过 ≥7 的情绪标签（记录在 DB 的 anchored_labels 字段）遗忘概率降至 5%。

#### 第三层：记忆锚点层（翻旧账复活）
情绪强度 ≥ 7 时自动记录锚点标签到 DB，并存入向量库作为情感锚点。
后续检索到时复活：对应情绪 + (锚点强度 × 0.4)，委屈 +1。

#### 第四层：基底情绪层（性格底线）
衰减后不低于基底值：醋意 0.8、撒娇 0.5、害羞 0.3、开心 0.2、生气/委屈 0.0。

#### 用户情绪分析
使用 DeepSeek-Chat（temperature=0），三步走：
1. **关键词触发**：预定义词表（"菜"→生气，"可爱"→害羞），命中直接返回
2. **LLM 分析**：结构化 prompt 输出 JSON，含 recent context 参考
3. **关键词兜底**：API 失败或置信度 < 0.5 时备用

#### 情绪感染映射
在 `compute_infection_updates()` 中执行，开头优先检查和解映射：

**和解映射（优先）**：
- "对不起""我错了""别生气" → 生气-3，委屈-3，撒娇+1
  - 委屈 ≥ 7 时替代效果：生气-3，委屈-1（下降减半），撒娇+1
- "不气不气""乖" → 生气-2，害羞+2
- "开玩笑的""逗你的" → 委屈-3，生气+1
  - 委屈 ≥ 7 时替代效果：委屈-1（下降减半），生气+1

**感染映射（未命中和解时）**：
用户情绪 → AI 情绪变化，按公式缩放：`min(基础量 × intensity × confidence × 0.4, 5)`

主要规则：
- 用户开心（夸AI）→ AI 害羞+3，开心+1
- 用户开心（分享好事）→ AI 开心+2；提到别人 → 醋意额外+1
- 用户生气（对AI不满）→ AI 委屈+3，生气+1，撒娇+1
- 用户生气（对其他人）→ AI 生气+1；有醋意 → 醋意-2
- 用户委屈 → AI 委屈+1，撒娇+1
- 用户害羞 → AI 害羞+4，开心+2
- 用户撒娇 → AI 撒娇+2，开心+1
- **反转抑制**：AI 生气 > 5 且用户示好（害羞/撒娇）→ 生气额外 -3

#### 情绪突变检测
AI 回复后，主导情绪 ≥ 6 时检测反转信号：
- 生气时回复含"笑死/哈哈/好吧算了/噗/服了/算了" → 生气 50% 转移给开心
- 委屈时含"其实还好/也没那么糟/我开玩笑的/没事啦" → 委屈 50% 转移给开心
- 醋意时回复含"骗你的/没事啦/你去吧/我才不" → 醋意反向 +2（嘴硬）

#### /status 状态查询
支持指令：`/状态`、`/status`、`知慧你现在怎么样`
- 截获消息后直接获取当前情绪向量（含半衰期衰减 + 基底保底）
- 通过 `emotion_to_natural()` 转为自然语言描述（含主导+次要情绪叠加）
- 跳过完整的 10 步情绪流程，直接回复并 return

### 图片识别系统

采用"收图不认图"策略，将图片识别延迟到回复决策之后：

1. **图片收集（无条件）**：遍历消息段 `type=="image"` 提取 URL，纯 Python 操作无开销
2. **编号占位符**：`[图片]` → `[图片1]` `[图片2]`，存入短期记忆作为上下文
3. **概率检查过滤**：群聊约 90% 的消息在此阶段被忽略，`recognize_image` 一次都不执行
4. **延迟并行识别**：确定回复后，使用 `asyncio.gather` 并行调用通义千问视觉模型
5. **上限保护**：`MAX_IMAGE_RECOGNITION=3`，防止刷图消耗 token

Agent 输入的 enriched_input 格式：
```
[图片内容：[图片1：一只橘猫在沙发上睡觉] [图片2：猫的特写]]
[图片1] [图片2] 你看这个猫可爱吗
```

#### 情绪注入方式
情绪上下文不再拼入 chat_history，而是通过 system message 注入：
```python
executor = get_agent_executor(emotion_context_str)
```
`get_agent_executor()` 工厂函数动态创建 AgentExecutor，不修改全局 prompt。
`personality.txt` 末尾追加了情绪-行为映射规则（害羞≥6短句省略号、生气≥5回复冷淡等）。

#### 好感度系统
- **范围**：0 ~ 10000，**新用户默认 2000**
- **7天半衰期**：好感度随真实时间衰减，公式 `factor = 0.5^(dt/604800)`
- **变化公式**：`delta = Σ(AI情绪强度 × 系数)`，正向/负向情绪自然拉动
  - 撒娇/害羞：+30/单位 | 开心/醋意：+15/单位 | 委屈：-15/单位 | 生气：-25/单位
  - 例如 AI 害羞=6+开心=4 → 6×30+4×15=+240，走心对话累积
- **后台漂移**：每 3 小时，>7500 减 100（低至 5000），<500 加 50（高至 2000）
- **影响回复概率**：好感度 0→0.3，10000→0.9，线性映射

### 共情系数（2025-04-27 新增）

好感度影响感染映射强度，模拟"高好感用户伤心时 AI 更心疼"：

```
empathy_scale = 1.0 + (affection - 2000) / 10000
```

| 好感度 | 共情系数 | 效果 |
|--------|---------|------|
| 0 | 0.8x | 低好感→情绪感染减弱 |
| 2000（基准） | 1.0x | 新用户默认，不变 |
| 5000 | 1.3x | 偏好感→轻度放大 |
| 10000 | 1.8x | 满好感→非常在意用户情绪 |

- 在 `compute_infection_updates()` 返回后执行，只放大正值 delta（增强情绪），负值不变（消退不缩放）
- `get_user_affection()` 在步骤4中调用，结果缓存后复用于步骤5的好感度更新，避免重复读库

### 情绪偏置检索 + 认知偏置指令（2025-04-27 新增）

**情绪偏置检索**：主导情绪 ≥ 5 时，额外检索同情绪标签的情感锚点记忆：

```python
bias_docs, bias_metas = vector_store.retrieve(
    session_id, raw_message, k=2, where={"emotion": dominant_emo}
)
```

- 偏置结果前置到常规检索结果之前，优先被锚点复活机制处理
- 无匹配时优雅降级（空列表），不影响常规流程
- ChromaDB metadata filter 原生支持，无需额外索引

**认知偏置指令**：情绪影响 LLM 对信息的解读倾向，通过 system message 的 `{emotion_context}` 注入：

| 主导情绪 | 注入的偏置指令 |
|---------|--------------|
| 委屈 | 更容易被模糊的表述触动，对忽视和冷落更加敏感 |
| 生气 | 更容易把中性信息解读为挑衅或冒犯 |
| 开心 | 对事物的看法更加积极和宽容 |
| 醋意 | 对风吹草动容易多想 |
| 害羞 | 容易不好意思说出真实想法 |
| 撒娇 | 更倾向于用软萌语气说话 |

情绪偏置检索 + 认知偏置 + 伪装策略三者协同效果示例：

```
用户说"你好敷衍"
  → 情绪分析: 委屈+3, 生气+1
  → 共情系数 1.6x（好感度8000）→ 委屈+4.8
  → 社交过滤器: 信任规则触发 → 跳过委屈抑制 → 保持脆弱表达
  → 偏置检索: 优先找回之前"被敷衍"的记忆片段
  → 认知偏置: "委屈感较强，对忽视更加敏感"
  → Agent 回复: "你上次也这么说…（委屈）"
```

#### 完整处理流程
```
 0. [截获] /status → 查情绪直接回复，return
 1. 获取情绪（半衰期衰减 + 基底保底）
 2. ≤30分钟？→ 快速衰减 + 遗忘（锚点保护）
 2b. 收集图片 URL（上限 3 张，编号占位符替换 [图片]→[图片1]...）
 3. LLM分析用户情绪
 4. 置信度≥0.5？→ 感染映射（先检查和解）
 4a. 共情系数缩放（好感度影响感染强度）
      → 更新AI情绪 → 锚点≥7存向量库
 5. 更新好感度（用户情绪 + AI自身情绪拉动）
 6. 检索长期记忆（+ 情绪偏置检索：主导情绪≥5时额外查同类锚点）
 6a. 伪装策略预检查（跳过抑制标签）
 6b. 社交过滤器（傲娇抑制，跳过已标记标签）
 6c. 伪装策略变换（醋意→冷漠映射）
 7. 情感锚点复活（翻旧账）
 8. 重读最终情绪
 8a. 认知偏置指令（主导情绪≥5时附加）
 9. 并行识别图片（仅当确定回复，asyncio.gather）
10. 构造 Agent 输入（含情绪上下文 + 认知偏置）
11. Agent 生成回复
12. 主导情绪≥6？→ 突变检测
```

### 消息分段发送

AI 生成的回复通过 `split_into_messages()` 按自然语义边界拆分：

1. 优先按 **双换行** 拆分
2. 其次按 **单换行**
3. 最后按 **句子边界** 拆分（正则匹配）：
   - `。！？` → 句号/感叹号/问号
   - `...` + 中文字符 → 省略号后接新句子
   - 颜文字结尾 `)）` + 中文字符 → 如 `(￣へ￣)不过` 会从颜文字后拆分

拆分后最多发送 `MAX_REPLY_MESSAGES`（默认 3）条。

**延迟策略**：
- 主延迟（`MAIN_DELAY`）6 秒
- 第一段无字符延迟
- 后续段落每字 100ms 延迟
- 段落间随机延迟 1~3 秒

### CQ码清理

`extract_clean_text()` 将 NapCat 消息段转为可读文本：

| 类型 | 转换结果 |
|------|---------|
| `text` | 原文 |
| `face` | `[表情:名称]` 或 `[表情]` |
| `image` | `[图片]` |
| `at` | `@QQ号` |
| `reply` | `[回复:内容]` |

### 记忆系统

- **短期记忆**：SQLite 存储最近 50 轮对话，达到阈值后自动总结存入长期记忆
- **长期记忆**：ChromaDB 向量存储，支持相似性检索
- **话题切换检测**：90 分钟无互动 或 Jaccard 相似度 < 0.3 视为新话题

### 人格设定

详见 `personality.txt`。核心特征：
- 16 岁 AI 少女，傲娇毒舌
- 被夸会脸红否认，占有欲强
- 回复简短口语化，禁止句末句号
- 准确回应：用户发什么就回应什么，不夸大
- 情绪强度影响语气（害羞→省略号/结巴，生气→冷淡，撒娇→黏人等）

## 社交过滤器 + 认知资源分配权重（2025-04-26 新增）

作为表达层包裹器，在真实情绪传递给 LLM 之前进行傲娇风格过滤。

### 核心机制

**认知资源分配权重（Cognitive Resource Allocation Weight）**：
- 每次抑制傲娇本能（害羞→否认、开心→嘴硬）消耗认知资源
- 资源随**现实时间**恢复（默认 1 单位/分钟）
- **好感度决定资源容器大小**：
  - 低好感 → 容器小(20) → 灌满快 → 宏观恢复快，但每次可用资源少 → 回复敷衍
  - 高好感 → 容器大(100) → 灌满慢 → 宏观恢复慢，但每次可用资源多 → 愿意深聊

**傲娇过滤规则**：

| 规则 | 条件 | 效果 | 资源消耗 |
|------|------|------|---------|
| 害羞抑制 | 害羞 ≥ 3 | 害羞 × 0.5，撒娇 +1 | 0.05 |
| 开心否认 | 开心 ≥ 4 且被夸 | 开心 -2，害羞 +1 | 0.08 |
| 醋意压抑 | 醋意 ≥ 3 | 醋意 × 0.6 | 0.06 |
| 委屈假装 | 委屈 ≥ 6 | 委屈 × 0.5 | 0.04 |

**爆发模式**：资源比例 < 20% 时抑制失败，真实情绪增强 1.5 倍表达（替代突变检测）。

**伪装策略规则**（2025-04-27 新增）：

在傲娇抑制基础上，根据好感度增加一层情绪变换，模拟"不熟时掩饰"和"信任时坦率"：

| 规则 | 条件 | 效果 | 资源消耗 |
|------|------|------|---------|
| 不熟_醋意掩饰为冷漠 | 醋意 ≥ 7 且 好感度 < 3000 | 醋意 × 0.3，40% 转为生气（冷漠） | +0.08 |
| 信任_直接表达脆弱 | 委屈 ≥ 6 且 好感度 > 7000 | 跳过委屈抑制规则，不假装没事 | -0.03 |

- 规则在抑制前检查（标记跳过标签）和抑制后执行（变换），两遍扫描
- "冷漠掩饰"在抑制后执行：醋意降到 30%，40% 映射到生气
- "直接表达"在抑制前检查：委屈标记跳过规则4，保持原值传递给 LLM
- 好感度从 `state.max_capacity` 通过 `_reverse_affection()` 反推，无需改调用签名

## 故障排除

### 最近修复

1. **`update_user_affection` 死锁导致不回复**（2025-04-26）
   - 问题：`update_user_affection` 在持锁状态下调用 `get_user_affection`，`asyncio.Lock` 不可重入导致死锁，情绪感染后所有逻辑被阻塞。
   - 解决：提取无锁同步版本 `_get_user_affection_no_lock`，内部调用不再重复加锁。

2. **图片识别漏细节**（2025-04-24）
   - 问题：图片识别 prompt 太泛，模型只描述主体忽略环境细节（如闪电）
   - 解决：`image_tools.py` prompt 改为明确要求描述"所有物体、动作、环境背景、天气现象等全部细节"

3. **情绪分析误判调侃为委屈**（2025-04-24）
   - 问题：用户的调侃/纠正语气被 LLM 误判为"委屈"
   - 解决：LLM prompt 增加区分要点（调侃→开心或中性，无奈→中性），新增两个 few-shot 示例

4. **CQ码导致回复内容错误**（2025-04-23）
   - 解决：添加 `extract_clean_text()` 将 CQ 码转可读文本

4. **回复未分段**（2025-04-23）
   - 解决：扩展拆分正则，支持 `...` + 中文 和 颜文字 `)` + 中文

5. **回复句末带句号**（2025-04-23）
   - 解决：personality.txt 添加"禁止在句末使用句号"规则

6. **字符延迟每段都加**（2025-04-23）
   - 解决：第一段跳过字符延迟，后续段落保留

### 历史问题

- 时间戳出现在 AI 回复中 → 修复
- 回复速度过快 → 添加字符数延迟
- 编码问题（Unicode/GBK） → 添加异常处理
- NapCat 连接超时 → 确认 webhook 地址为 WSL 内网 IP
